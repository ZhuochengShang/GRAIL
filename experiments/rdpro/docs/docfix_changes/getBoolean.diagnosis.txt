ROOT CAUSE:
The documentation instructs users to create the options object using `OperationHelper.parseCommandLineArguments`, which triggers classloading of Hadoop/Spark components that require Jetty dependencies (`org/mortbay/jetty/handler/AbstractHandler`) not present on the standalone test classpath. The `BeastOptions` class can and should be instantiated directly for testing.

```scala
val opts = new edu.ucr.cs.bdlab.beast.common.BeastOptions(false)

val v1: Boolean = opts.getBoolean("option-4", defaultValue = false)
val v2: Boolean = opts.getBoolean("missing-option", defaultValue = true)

require(v2 == true, s"Expected missing-option fallback=true but got $v2")
```

edu.ucr.cs.bdlab.beast.common.BeastOptions (Scala)
