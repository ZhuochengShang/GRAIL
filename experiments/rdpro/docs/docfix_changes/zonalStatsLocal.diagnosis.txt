ROOT CAUSE:
The snippet failed because it attempted to extract a local `IRasterReader` from a distributed `RasterRDD` using a non-existent property (`rasterRDD.rasterFeature`). `zonalStatsLocal` is a low-level API that bypasses Spark's distributed processing; it requires an explicitly initialized local `IRasterReader` pointing directly to the file path, which cannot be trivially unwrapped from a distributed RDD of tiles.

CORRECT USAGE:
```scala
val zones: Array[edu.ucr.cs.bdlab.beast.geolite.IFeature] = featuresRDD.take(32)
require(zones.nonEmpty, "No zones available for zonalStatsLocal test")

// Initialize the local raster reader provided by the harness
val path = new org.apache.hadoop.fs.Path(rasterFile)
val fs = path.getFileSystem(sparkContext.hadoopConfiguration)
rasterReader.initialize(fs, rasterFile, "0", new edu.ucr.cs.bdlab.beast.common.BeastOptions())

// Execute local single-threaded zonal statistics
val zsResults: Array[edu.ucr.cs.bdlab.raptor.Collector] = 
  edu.ucr.cs.bdlab.raptor.ZonalStatistics.zonalStatsLocal(
    zones, 
    rasterReader, 
    classOf[edu.ucr.cs.bdlab.raptor.Statistics]
  )

val total = zsResults.length
val nonNull = zsResults.count(_ != null)
require(total == zones.length, s"Result length $total does not match zones length ${zones.length}")
```

DOC MUST SAY:
- `zonalStatsLocal` requires a local `IRasterReader` initialized directly from the file system and path (e.g., using `rasterReader.initialize(...)`).
- Do not attempt to extract a reader from a distributed `RasterRDD` (e.g., `rasterRDD.rasterFeature` does not exist and will fail to compile).
- The harness provides an uninitialized `rasterReader` and `rasterFile` (path string) which must be used to initialize the reader before calling `zonalStatsLocal`.
