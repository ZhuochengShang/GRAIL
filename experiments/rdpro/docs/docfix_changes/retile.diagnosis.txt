ROOT CAUSE:
The documentation incorrectly demonstrates calling `retile` on a `RasterRDD`, which fails at runtime with a Spark SQL serialization error (`Unrecognized value [F` for `Array[Float]`) because `ITile` extends `Row` and Catalyst cannot natively encode raw float arrays during shuffle/evaluation. As explicitly stated in the receiver type hint, `retile` must be called as an instance method on `RasterMetadata`, which can be extracted from the RDD's tiles.

CORRECT USAGE:
```scala
val metadata = rasterRDD.first().rasterMetadata
val retiledMetadata = metadata.retile(64, 64)
```

REQUIRED TYPES & IMPORTS:
edu.ucr.cs.bdlab.beast.geolite.RasterMetadata (Scala class)

DOC MUST SAY:
- `retile` must be called as an instance method on `RasterMetadata`, not directly on a `RasterRDD`.
- A `RasterMetadata` instance can be obtained from a `RasterRDD` by accessing the metadata of its first tile (e.g., `rasterRDD.first().rasterMetadata`).
- Calling `retile` directly on a `RasterRDD` containing float pixels will cause a Spark SQL runtime serialization error (`Unrecognized value [F`) when the RDD is evaluated.
