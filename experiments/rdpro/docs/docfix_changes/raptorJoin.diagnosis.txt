ROOT CAUSE:
The documentation incorrectly implies that the pixel value `m` in the join result is always a scalar (e.g., `v.m >= 0`), but in RDPro, multi-band rasters yield an array of values (e.g., `Array[Float]`, represented at runtime as `[F`). Because the snippet called the Java helper without explicit type parameters and passed `m` to `java.lang.Float.isNaN`, Scala's type inference incorrectly inferred `T = Float` and inserted a cast. At runtime, the underlying `Array[Float]` could not be cast to `java.lang.Float`, throwing a `ClassCastException`. Furthermore, the snippet ignored the instruction to use the Scala implicit method `raster.raptorJoin(vector)`.

CORRECT USAGE:
```scala
// Use the Scala implicit method directly on the RasterRDD
val joined = rasterRDD.raptorJoin(featuresRDD, new BeastOptions())
val n = joined.count()
require(n > 0, "empty result for raptorJoin")

val sample = joined.first()
// Safely handle the pixel value which may be an array (multi-band) or scalar (single-band)
val mFirstBand = sample.m match {
  case arr: Array[Float] => arr.headOption.getOrElse(Float.NaN)
  case f: Float => f
  case _ => Float.NaN
}

require(!java.lang.Float.isNaN(mFirstBand) && !java.lang.Float.isInfinite(mFirstBand), s"invalid pixel value in raptorJoin result: $mFirstBand")
println("__CHECK__ raptorJoin " + s"count=$n,m=$mFirstBand")
```
