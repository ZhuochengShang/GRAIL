ROOT CAUSE:
The documentation and snippet incorrectly assume the type parameter `T` should be a scalar primitive (e.g., `Float` or `Int`) when the underlying raster data is actually an array of primitives (e.g., `Array[Float]`). The runtime error `Unrecognized value [F@... of type class [F` occurs because `[F` is the JVM signature for `float[]` (`Array[Float]`), meaning the test harness provided a `RasterRDD[Array[Float]]` but the snippet typed it as `RasterRDD[Float]`, causing a `ClassCastException` / unrecognized type error when `MemoryTile` attempts to process the pixel values.
