ROOT CAUSE:
The method `addTile` on the `AbstractConvolutionTile` class is defined with `private[raptor]` visibility. This restricts its access to code within the `edu.ucr.cs.bdlab.raptor` package. The documentation incorrectly presents this as a public API, and any user code attempting to call it from outside this specific package will fail with a compilation error.

CORRECT USAGE:
```scala
// This code will only compile if it is placed inside the `edu.ucr.cs.bdlab.raptor` package.
import edu.ucr.cs.bdlab.beast.geolite.{RasterFeature, RasterMetadata}
import java.awt.geom.AffineTransform

// 1. Create a source tile to be added
val metadata = new RasterMetadata(0, 0, 100, 100, 10, 10, 4326, new AffineTransform())
val sourceTile = new MemoryTile[Float](0, metadata, RasterFeature.create(Array("fileName"), Array("testFile.tif")))
sourceTile.setPixelValue(5, 5, 100.0f)

// 2. Create the convolution tile, passing the source tile's ID as the `stid`
val weights = Array.fill[Float](9)(1.0f / 9.0f) // 3x3 averaging kernel
val convTile = new ConvolutionTileSingleBand(0, metadata, sourceTile.rasterFeature, 1, weights, sourceTile.tileID)

// 3. Call the internal addTile method
convTile.addTile(sourceTile)
```

DOC MUST SAY:
*   This method is `private[raptor]` and is not part of the public API. It is intended for internal use within the library.
*   Directly calling this method from user application code is not supported and will result in a compilation error.
*   The `ConvolutionTile` and its methods are low-level components used by higher-level distributed operations like raster convolution. Users should not construct or manipulate these tiles directly.
