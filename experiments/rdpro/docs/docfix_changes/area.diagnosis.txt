The source code shows `area` is an instance method on the concrete class `edu.ucr.cs.bdlab.davinci.LiteList`, not its abstract parent `LiteGeometry`. A user cannot call this method without first constructing a `LiteList` instance, which requires providing coordinate arrays of type `Array[Short]`, a critical detail omitted from the documentation.

// Construct the required receiver object, an instance of LiteList.

// The area method can now be called on the LiteList instance.
