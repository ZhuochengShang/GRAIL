ROOT CAUSE:
The method `saveAsKML` is an implicit extension method on a standard Spark `RDD[IFeature]`, made available by importing members of the `ReadWriteMixin` object. The documentation incorrectly guided the user to convert their `RDD` to a `JavaSpatialRDD`, which is a wrapper intended for the Java API. This led to a compile error because `JavaSpatialRDD` is a class, not an object, and has no static `create` method as the snippet attempted to use.

CORRECT USAGE:
```scala
// Import the implicit conversions from ReadWriteMixin to enable the .saveAsKML extension method
import edu.ucr.cs.bdlab.beast.io.ReadWriteMixin._

// Assuming 'featuresRDD' is an existing RDD[IFeature]
val outputPathString = s"$output_dir/features.kml"

// Call the extension method directly on the RDD[IFeature]
featuresRDD.saveAsKML(outputPathString)
```

DOC MUST SAY:
*   For Scala users, `saveAsKML` is an extension method on `RDD[IFeature]`.
*   To use this method, you must import the implicit conversions from the `ReadWriteMixin` object: `import edu.ucr.cs.bdlab.beast.io.ReadWriteMixin._`.
*   The receiver of the call should be a standard `RDD[IFeature]`, not a `JavaSpatialRDD`.
*   The documentation should clearly separate the idiomatic Scala usage (extension method on `RDD`) from the Java-specific helper functions (static methods on `JavaSpatialRDDHelper`).
