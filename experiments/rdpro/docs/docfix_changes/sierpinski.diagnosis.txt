ROOT CAUSE:
The documentation for `sierpinski` is misleading. It states that the `cardinality` parameter is "the number of records to generate". However, the runtime error shows that calling `sierpinski(1000)` produces only 500 records. The underlying implementation of the `SierpinskiDistribution` generator produces `cardinality / 2` records, a critical semantic detail omitted from the documentation.

val generatedRDD: edu.ucr.cs.bdlab.beast.SpatialRDD = sc.generateSpatialData.sierpinski(2000)
val count = generatedRDD.count()
require(count == 1000)
