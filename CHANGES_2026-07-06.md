- `dedup_report()` + CLI **`aideal dedup [--out …] [--full]`** — deterministic
  redundancy audit: overload collapse (canonical site election), forwarder
  alias edges (full = every site forwards → collapses catalog entry; partial
  kept: `shapefile`'s Java-context site forwards to `spatialFile` but the
  Scala mixin site is the documented entry point), same-signature twins,
  alias-registry cross-check (`aliases.json` + ``(alias for `X`)`` scan).
- `llm_common_apis()`: an EMPTY cached `intent_common.json` is now a cache
  miss, not a permanent "no common APIs" verdict (the RDPro cache was `[]`).
- `_subsume_overloads()` — telescoping-overload rule ("needed longest
  parameters"): a same-file overload whose param types are a PREFIX of a
  longer sibling's is the same function with defaults → catalog keeps only
  the longest signature; short forms recorded as `subsumed_overloads`
  (RDPro: 19 sites / 17 names). Canonical election: non-deprioritized path >
  documented > longest params. New `codebase.dedup.deprioritize_paths`
  (adapter default `Java*.scala`) keeps Java facade duplicates from winning —
  without it geoTiff's canonical was `JavaSpatialSparkContext…: JavaRasterRDD`,
  the exact form the pinned preamble avoids; with it, canonicals land on
  `RaptorMixin.geoTiff` / `RaptorMixin.raptorJoin` / `ReadWriteMixin.shapefile`
  (the verified preamble call forms). Readme generation uses the same
  election: `overloads` lists only genuinely distinct signatures, and
  Scaladoc falls back to the best-documented family member.

## Full-run reliability (why the 2026-07-01 all-API run died — all fixed)

Diagnosis from `run_20260701_110426.log`: the run CRASHED ~120/218 APIs in
with `TypeError: can't concat str to bytes` — `subprocess.TimeoutExpired`
returns the captured streams as BYTES even under `text=True`, and `plotImage`
hit the 600s timeout. All per-API results were lost (only error_log rows
survived); `docs/comprehension_run.json` is the corrupt remnant.

`src/aideal/doc_checks.py`:
- timeout handler decodes bytes defensively (the crasher);
- LLM calls in the attempt loop are guarded — provider errors (quota /
  network / bad model id) record `fail [llm-error]` for that API and move on;
- per-API JSONL checkpoint (`.aideal_exec/comprehension_progress.jsonl`),
  flushed as each API finishes; **`--resume`** skips finished APIs and
  pre-fills their results (verified: resumed run makes 0 LLM calls for them);
- **`--timeout N`** per-attempt override (visualization APIs are the slow
  ones: 17 viz-ish names are selected; 600s × 4 attempts worst case each).
