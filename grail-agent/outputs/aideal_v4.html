LLM model
Integration test (puzzle eval) settings
K — functions per draw
?
K functions are randomly picked from your API and combined into one joint program. K=5 means each test must use exactly 5 functions together — testing how they compose.
N — number of draws
?
N is how many random K-function programs are generated and tested. N=50 = 50 different random combinations. Pass rate = programs that compile and pass ÷ 50. Higher N = more reliable signal, higher LLM cost (~$0.02/draw).
Each draw = random K functions → LLM writes one joint program → compile + test
Pass threshold
?
If integration test pass rate falls below this, the CI check fails and the PR is blocked. 80% = at least 40/50 draws must pass.
